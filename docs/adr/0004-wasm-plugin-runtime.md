# ADR-0004: WASM plugin runtime — wasmtime + the Component Model

- **Status:** Accepted
- **Date:** 2026-06-27
- **Deciders:** Maintainer, with `plugin-systems-engineer`, `security-engineer`

## Context

Cairn's extensibility (PRD §7.8) lets third parties add backends, viewers, and actions. Plugins are
untrusted and must be safe-by-default: no ambient filesystem/network, brokered credential access
only, resource-limited so a bad plugin cannot hang the UI. We need a typed, versioned, language-
agnostic host/guest contract.

## Decision

Use **`wasmtime` + the WebAssembly Component Model / WIT**, with host bindings generated by
`wit-bindgen`. A plugin's exported `backend` world is wrapped host-side in a `PluginVfsBackend` that
implements `Vfs`, so plugin backends are indistinguishable from built-ins. Capabilities are
default-deny (the `Linker` only exposes granted host functions); WASI ambient fs/sockets/process/env
are withheld; all I/O is brokered and journaled. Credentials are accessed via opaque session-scoped
token stand-ins the host substitutes at dispatch (the plugin never sees the value). Isolation via
per-instance `Store`, `ResourceLimiter` memory caps, and epoch-interruption timeouts; calls run off
the render path. Streaming uses chunked polling now (page-token loops; `read-chunk`/`write-chunk`
resource methods) with native WIT `stream<T>` adopted additively when stable. The WIT world is
semver'd: major = breaking (refused), minor = additive.

## Consequences

### Positive
- Typed, self-describing, versioned ABI; plugin authors can use any language with a component
  toolchain.
- Strong sandbox: capability-scoped, no ambient authority, brokered secrets, hard resource limits —
  same trust boundary as the AI layer.
- Mature, well-audited runtime with first-class async.

### Negative / trade-offs
- Native WIT async streams are still stabilizing → chunked-polling shim now (clean migration path).
- `Store: Send` constraint requires `Send` host data and careful resource-destructor registration.
- Per-page WIT round-trips add overhead → plugins should return ≥100-entry pages.

## Alternatives considered
- **Core-wasm + hand-rolled ABI** — unmaintainable past a few functions, Rust-only guests; rejected.
- **`extism`** — bytes-in/out function model; wrong shape for multi-method backends/viewers; rejected.
- **`wasmer`** — weaker governance and license-stability history vs wasmtime; rejected.
